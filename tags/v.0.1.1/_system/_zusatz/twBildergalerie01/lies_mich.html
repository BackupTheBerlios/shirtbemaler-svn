Installation

Nach Entpacken des Zip-Archives sollte obige Verzeichnisstruktur vorhanden sein. Nachdem diese (ohne die lies_mich.html) auf den Webspace mittels FTP hochgeladen wurde, müssen ggf. in der Datei config.php die Angaben im ersten Abschnitt bezüglich der Pfade angegeben werden. Zum Editieren aller Dateien eignet sich ein einfacher Texteditor. Textverarbeitungsprogramme sind dazu ungeeignet.
Wichtig ist dem/n Verzeichnis(sen) im Verzeichnis [bilder], in die die Bilddateien gespeichert werden sollen die entsprechenden Lese-, Schreib- und Ausführungsrechte zu geben (chmod 777). Das ist wichtig, da das Skript einen Unterordner [thumbs] anlegen soll, in den dann die Thumbnails für diesen Ordner erstellt werden.
Dafür sind entsprechende Zugriffsrechte notwendig. Sollte auf Ihrem Webspace Safe Mode aktiv sein, wird es wohl notwendig sein, auch diesen Unterordnern manuell chmod 777 zuzuweisen.
Danach sollte die Galerie eigentlich schon einsatzbereit sein.

Anpassen des Designs

Im zweiten Abschnitt der config.php werden Variablen Werte zugeordnet, die in der index.php, bzw. der frame.php als CSS umgesetzt werden. Die Kommentare in der config.php erklären, welcher Wert für welches Element Wirkung zeigt. Tiefergehende Änderungen können aber auch direkt in den Dateien index.php und frame.php vorgenommen werden, sofern dies nötig erscheint.
Die Grafik default.jpg wird geladen, wenn die Galerie ohne Parameter aufgerufen wird. Die Ausmasse dieser Grafik sind abhängig von den Angaben in der config.php bezüglich der Dimensionen des Iframes. Bei der Gestaltung können Sie Ihrer Fantasie freien Lauf lassen.

Bilder hinzufügen

Abhängig von den in der config.php angegebenen Masse für den Iframe können Sie Ihre Bilder entsprechend mit einem Grafikprogramm skalieren. Wenn die Bilder grösser ausfallen, als der Iframe, erscheinen Bildlaufleisten am Iframe. Sofern das nicht als störend empfunden wird, muss darauf keine Rücksicht genommen werden.
Die Bilder müssen dann entsprechend in das gewünschte Verzeichnis hochgeladen werden. In dem Verzeichnis mit diesen Bildern muss dann eine Datei bilderliste.txt vorhanden sein, in der die Bilder gelistet werden und wenn gewünscht auch Text angegeben werden kann. Dabei ist nach folgendem Schema zu verfahren:
bildname.[jpg|gif|png] ++ Text
Beachten Sie bitte, dass hier zwischen Gross- und Kleinschreibung unterschieden werden muss!
Die Reihenfolge in dieser Liste entscheidet dabei auch über die Reihenfolge der Thumbnails im Anzeigebereich. Wenn Sie diese Schritte vollzogen haben, rufen Sie dieses Album im Browser auf. Die Thumbnails werden nun automatisch erstellt, sofern die Vorraussetzungen bezüglich der Lese-, Schreib- und Ausführungsrechte stimmen und eine GD-Lib auf ihrem Websace installiert ist. Dieser Vorgang kann einige Zeit in Anspruch nehmen, je nachdem wieviele neue Thumbnails zu erstellen sind. Diese werden dann im Unterverzeichnis [thumbs] gespeichert, so daß beim nächsten Aufruf die Thumbnails direkt verfügbar sind.

Bilderalben editieren oder hinzufügen

Im Verzeichnis [bilder] können Unterverzeichnisse als Bilderalben angelegt werden. Die Informationen über diese Verzeichnisse sind in der Datei alben.txt gespeichert. Auch hier können sie die Information angeben, indem sie diese Datei editieren, nach folgendem Schema:
verzeichnisname ++ Text
Beachten Sie bitte, dass hier zwischen Gross- und Kleinschreibung unterschieden werden muss!
Geben Sie diesem Verzeichnis entsprechende Zugriffsrechte (chmod 777) und legen sie eine neue Datei bilderliste.txt an. Tragen Sie in diese Liste die Bilder in diesem Verzeichnis ein und speichern Sie diese Bilder in dem Verzeichnis auf Ihren Webspace. Rufen Sie dieses Album dann einmal im Browser auf, um die Thumbnails erstellen zu lassen.

Urheberrecht

Das Skript darf von jedem ohne Einschränkungen benutzt werden. Ich würde mich aber freuen, wenn der kleine Backlink auf meine Homepage nicht entfernt würde. Auf jeden Fall bitte ich darum folgenden Metatag in der XHTML Vorlage zu lassen, bzw. in eine neue Vorlage im head Bereich zu übernehmen:

Auch das Löschen der Kommentarzeilen in den Skriptdateien ist nicht zulässig. Sollten Teile des Codes für eine eigene Weiterentwicklung dienen, ist das in Ordnung. Ungemein ungehalten wäre ich jedoch, dieses Skript mit fremden Copyrightvermerken wiederzuerkennen.
